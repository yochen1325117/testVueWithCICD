使用Travis-CI构建Android项目并自动打包部署到GitHub-Release

如何部署至GitHub Pages？

.travis.yml

nuxt 建立

npx create-nuxt-app project-name

github 與 travic 連接

travic 執行檔 .travis.yml(只會動的那種)

          language: node_js
      node_js:
          - node
      install:
          - npm install
      script:
          - npm run build
          - npm run generate

github page 連結

github -> project page -> setting -> page

nuxt 與 github page

github page 使用的是靜態檔案

nuxt 的靜態檔案需要build -> generate後存後在dist中

若直接連接github page https://yochen1325117.github.io/testVueWithCICD/ 會是404

https://yochen1325117.github.io/testVueWithCICD/dist 才會是頁面

因此在 .travis.yml 添加 deploy 將 dist 資料夾推到某個tranch

再將github page 的brach指向dist的page

.travis.yml 加入以下指令 在最後面

        deploy:
          provider: pages
          skip-cleanup: true
          github-token: $GITHUB_TOKEN_FOR_TRAVIS
          target-branch: pageBranch
          local-dir: dist
          on:
              branch: master

skip-cleanup: travis 部屬時會清空dist，下skip-cleanup避免dist被清空

github-token: 先至 Settings/Developer settings/Personal access tokens 生一個給travis用的token

target-branch: 準備推到哪個branch

local-dir: 準備推什麼進去

on: 這個deply需要在哪個branch部署時執行

建好dist檔案 -> 把dist 推到 “pageBranch”(自己取的)
